Key Themes Across All Chapters

Central Idea

Programming languages make design choices at every level — from how names are resolved to how types are checked to how methods are dispatched. Each choice involves trade-offs between writability, readability, reliability, and efficiency. The same four criteria from Chapter 1 resurface throughout every subsequent chapter.

The Static/Dynamic Axis

One of the most consistent themes across all chapters is the trade-off between static (compile-time) and dynamic (runtime) approaches:

  • Static: determined at compile time — more efficient, more reliable, less flexible.
  • Dynamic: determined at runtime — more flexible, higher overhead, harder to debug.
  • Most modern languages use a mix (statically typed but with dynamic dispatch, e.g. Java).

Implementation Questions

The exam has one Scheme (Chapter 15) and one Prolog (Chapter 16) implementation question, both involving list processing. Practice is essential — understanding the concepts is not enough, you must be able to write correct code under time pressure in the actual interpreter.
