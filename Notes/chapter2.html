Chapter 2

Evolution of Programming Languages

The history of programming languages is not arbitrary — each language was shaped by the problems of its time, the hardware available, and the software engineering practices in vogue.

FORTRAN LISP COBOL ALGOL Pascal C Ada Smalltalk C++ Java

What You Need to Know for Each Language

⚠ Exam Framework

For each language in this chapter, you must be able to answer three questions:

  1. What was the environment? (intended users, application area, software engineering approach of the time, computer characteristics)
  2. How did the environment shape the language?
  3. What is the language's main contribution?

You do NOT need to know specific version numbers, exact dates, developers' names, or model numbers of computers.

Plankalkül (Zuse, ~1945)

Environment
Designed in isolation during WWII Germany, with no working computer to run it on. Intended for engineering calculations.
Contribution
The first high-level programming language ever designed, though it was never implemented at the time. Included assignment, loops, subroutines, and even something like floating-point numbers and arrays — remarkable given it predated all other HLLs. Largely unknown to the Western computing community until rediscovered decades later.

FORTRAN (FORmula TRANslation, 1957)

Environment
Scientific computing on large mainframes. Users were scientists and engineers who valued numerical computation. Computers were expensive and slow; efficiency was paramount. The dominant concern was: could a compiled high-level language produce code as efficient as hand-written assembly? No high-level language had been proven practical before FORTRAN.
Effect of environment on language
Designed for efficiency above all else. Arrays and floating-point operations were built-in primitives. The language had few safety features (no type checking, no bounds checking) because any overhead was unacceptable to its users.
Main contribution
The first successfully implemented, widely used high-level language. Proved that compiled code could be efficient enough for scientific work. Established that programming languages could be practical — a foundational moment in computing. Introduced the concept of a compiler as a practical tool.

LISP (LISt Processing, 1959)

Environment
Artificial intelligence research at MIT. AI required manipulating symbolic expressions, not just numbers. Researchers needed flexibility to represent and process knowledge structures dynamically.
Effect of environment on language
Designed around linked lists as the primary data structure, since AI problems involve symbolic structures of variable and unpredictable shape. Dynamic memory allocation (garbage collection) was essential because the size of data structures could not be predicted at compile time. Programs are data — code and data have the same form (S-expressions).
Main contribution
First functional programming language. Introduced garbage collection, dynamic typing, recursive functions as a natural programming style, and treating code as data. These ideas are foundational to functional and dynamic languages today (Scheme, Clojure, Python's lambda, JavaScript's first-class functions).

COBOL (COmmon Business-Oriented Language, 1960)

Environment
Business data processing (payroll, inventory, accounting). Intended users included non-programmer managers who needed to read and verify program logic. Required decimal arithmetic (not binary floating point), file processing, and report generation. The US Department of Defense wanted a common language that businesses could use across different computer vendors.
Effect of environment on language
Designed for readability by non-programmers: verbose English-like syntax (e.g., MULTIPLY PRICE BY QUANTITY GIVING TOTAL). Required separate DATA DIVISION to declare all data structures. Hierarchy records for nested data (predecessor of structs/records). Strong support for file I/O and formatted reports.
Main contribution
First language designed with readability as the primary criterion, not efficiency. Introduced the concept of hierarchical records (nested structured data). Demonstrated that languages could be designed for non-specialist users. Still processes a large fraction of the world's financial transactions today.

ALGOL 60 (ALGOrithmic Language, 1960)

Environment
Designed by an international committee of computer scientists to be a universal language for publishing algorithms and for scientific computation. Intended to improve on FORTRAN by being cleaner and more mathematically sound.
Effect of environment on language
Academic and committee-designed, so it prioritised elegance and conceptual clarity. Block structure emerged naturally as a way to scope variables. Recursive procedures were supported because the language was designed for describing algorithms, and recursion is a natural algorithmic tool.
Main contribution
Introduced block structure (nested scopes), recursive subprograms, and the concept of formal syntax description (Backus-Naur Form, BNF). Hugely influential on almost all subsequent imperative languages (Pascal, C, Java are all "ALGOL-like"). Introduced pass-by-name parameter passing. Never widely used in practice (no I/O, poor compiler support) but enormously influential on language design.

PL/I (1965)

Environment
IBM designed PL/I to replace both FORTRAN (scientific) and COBOL (business) with a single language. The intent was to eliminate the need for separate scientific and business languages.
Effect of environment on language
Tried to include everything from both worlds — the result was an enormous, complex language with many overlapping features. Almost every design decision from both predecessors was included.
Main contribution
First language to include exception handling (though crude). First to allow concurrency primitives. Demonstrated the danger of trying to do everything: a large, complex language that is difficult to learn, compile, and use correctly. A cautionary tale in language design (too little orthogonality, too much complexity).

SIMULA 67

Environment
Developed in Norway for simulation of discrete-event systems (e.g., queuing systems, networks). Programmers needed to model real-world entities with their own state and behaviour running concurrently.
Effect of environment on language
Naturally led to the concept of objects: each simulated entity had its own data (state) and procedures (behaviour). Co-routines were included for concurrent simulation. Classes were introduced as a way to describe the type of a simulated entity.
Main contribution
Invented the concept of classes and objects — the foundation of object-oriented programming. Introduced inheritance (subclasses). Every OOP language (Smalltalk, C++, Java, Python) traces its heritage to SIMULA 67. This is one of the most influential language innovations in history.

ALGOL 68

Environment
Successor to ALGOL 60, designed by committee to be a complete, rigorous, and orthogonal language.
Contribution
Extreme orthogonality: any construct could appear in any context. This consistency made the language conceptually elegant but very hard to learn and implement. Contributed the idea of user-defined data types and influenced Pascal and C. Largely unused in practice but highly influential on type theory.

Pascal (1971)

Environment
Designed for teaching structured programming. The goal was a small, clean language that could be easily learned and would encourage good programming habits (no goto, structured control flow). Computers were still relatively expensive; Pascal had to be implementable on small machines.
Effect of environment on language
Small and consistent; strongly typed; encouraged structured programming idioms. Strict type system prevented many common errors (e.g., no implicit conversions between numeric types). Limited I/O and file handling (designed for teaching, not production systems).
Main contribution
Enormously successful as a teaching language. Demonstrated that a small, clean, strongly-typed language was practical. Directly influenced Modula-2, Ada, and Delphi. Popularised structured programming among students and educators worldwide through the 1970s and 1980s.

C (1972)

Environment
Systems programming language designed to write the UNIX operating system. Users were expert programmers needing direct hardware access. Required portability (UNIX had to run on multiple machines), efficiency (close to assembly speed), and low-level memory control.
Effect of environment on language
Deliberately low-level: pointers, manual memory management, bitwise operations, direct memory access. Weak type checking (by design) to allow flexible casting and pointer manipulation. Compact and expressive for systems-level code. Portability was achieved through a thin hardware abstraction layer.
Main contribution
The dominant systems programming language — the language that UNIX and most operating system kernels, embedded systems, and interpreters are written in. Influenced almost every later language (C++, Java, C#, Python, JavaScript all borrowed syntax). Demonstrated that a high-level language can be efficient enough to replace assembly for systems work.

Prolog (1972)

Environment
AI research, especially natural language processing and theorem proving. Needed a language to express logical relationships and prove conclusions from facts, without specifying how the computation should proceed.
Contribution
First logic programming language. Programs are facts and rules; the interpreter uses backward chaining with backtracking to answer queries. See Chapter 16 for full details.

Ada (1983)

Environment
The US Department of Defense (DoD) needed a single language for all its embedded and real-time systems (weapons, aircraft, ships). Hundreds of different languages were in use, causing enormous maintenance costs. The software crisis of the 1970s had shown that large software systems were unreliable and expensive. The DoD needed a language that supported: large-scale software engineering, strong reliability, real-time systems, and concurrency.
Effect of environment on language
Very large and complex language with strong reliability features: strict strong typing, exception handling, built-in concurrency (tasks), packages for encapsulation, generics. Reliability was more important than writability or simplicity — the DoD was paying for reliability with maintenance cost of learned complexity.
Main contribution
Pioneered packages (encapsulation / data abstraction), generics, concurrent programming (tasking model), and strong exception handling in a compiled, strongly-typed language. Ada95 later added OOP. Widely used in aerospace, defense, and safety-critical systems to this day.

Smalltalk (1972–1980)

Environment
Research at Xerox PARC aimed at creating an educational computing environment. The goal was to make computing accessible to children and non-experts through a graphical, interactive system. Influenced by SIMULA's objects and LISP's dynamic nature.
Effect of environment on language
Everything is an object — including integers, classes, and even methods. Computation happens entirely through message passing. Dynamic typing allows flexibility. The system is self-describing and supports dynamic modification.
Main contribution
The first purely object-oriented language. Introduced the graphical user interface (GUI) paradigm. Defined the vocabulary of OOP: objects, messages, classes, inheritance, encapsulation, polymorphism. Directly influenced every OOP language that followed, including Java, Python, Ruby, and Objective-C.

C++ (1985)

Environment
Systems and applications programming. C programmers needed OOP features but could not afford to leave C's efficiency and existing codebases behind. There was enormous pressure to add OOP to C without breaking existing C code.
Effect of environment on language
Designed as a superset of C: all valid C programs are valid C++ programs (approximately). OOP features (classes, inheritance, virtual functions) were added on top of C's procedural foundation. No garbage collection — manual memory management preserved C's performance characteristics. Templates added for generic programming.
Main contribution
Brought OOP to the mainstream of systems and applications programming. Demonstrated that OOP and efficiency are compatible. Introduced templates (a powerful form of generic programming) and destructors for RAII. Its enormous success (and complexity) drove interest in cleaner OOP alternatives (Java, C#).

Java (1995)

Environment
The internet era. Software needed to run on any platform without recompilation (write once, run anywhere). Security was a major concern since code could be downloaded from the internet and executed. The complexity and unsafety of C++ (memory management, pointer arithmetic) were seen as liabilities for large-scale software development.
Effect of environment on language
Compiled to bytecode (JVM), not native code, so the same binary runs on any platform with a JVM. Garbage collection removed manual memory management. No pointer arithmetic for safety. Strong static type system. Built-in support for threading. Applets allowed code to run inside web browsers.
Main contribution
Platform independence through the JVM. Popularised garbage-collected OOP for mainstream development. Brought strong type safety and exception handling to a widely-used language. Enormously influential on C# and subsequent JVM languages. Made OOP the dominant paradigm in enterprise software development.

Perl, JavaScript, Python, Ruby

Environment: Text processing and system administration on Unix. Needed string manipulation, file handling, and regular expressions in a more convenient form than shell scripting.

Contribution: Powerful text processing with built-in regular expressions. Pragmatic "there's more than one way to do it" philosophy. Influenced PHP and modern scripting languages. Proved that scripting languages could be powerful general-purpose tools.

Environment: Web browsers needed a scripting language for client-side interactivity. Designed in ten days under time pressure to ship with Netscape Navigator.

Contribution: The language of the web browser — every major website depends on it. Dynamic typing, prototype-based OOP, first-class functions and closures. Despite its rushed origins, became one of the most widely used languages. Node.js extended it to server-side programming.

Environment: Designed to be readable, clean, and easy to learn. Intended as a teaching language and rapid-development tool. Emphasis on code readability as a primary value.

Contribution: Made readability a core design principle (significant indentation). Multi-paradigm (OOP + functional). Became the dominant language for data science, machine learning, scripting, and introductory programming education.

Chapter 2 Summary — Key Contributions at a Glance

  • Plankalkül: First HLL ever designed (never implemented until later).
  • FORTRAN: First successful compiled HLL; proved compilers can produce efficient code.
  • LISP: First functional language; garbage collection; code-as-data.
  • COBOL: First language prioritising readability; hierarchical records; business data.
  • ALGOL 60: Block structure; recursion; BNF syntax description; ancestor of C/Java.
  • PL/I: First exception handling; cautionary tale of over-complexity.
  • SIMULA 67: Invented classes, objects, and inheritance — OOP origins.
  • Pascal: Teaching language; small, clean, strongly typed; structured programming.
  • C: Systems programming; efficiency + portability; ancestor of most modern syntax.
  • Prolog: First logic programming language; backward chaining; AI focus.
  • Ada: Packages, generics, concurrency, exception handling; DoD reliability focus.
  • Smalltalk: Pure OOP; message passing; GUI pioneer; defined OOP vocabulary.
  • C++: OOP + C compatibility; templates; demonstrated OOP and efficiency coexist.
  • Java: Platform independence (JVM); garbage collection; safe OOP for the internet era.
