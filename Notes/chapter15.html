Functional Programming & Scheme

Functional programming computes by evaluating expressions, not executing statements. Scheme, a dialect of LISP, is the implementation language for this course. You must be able to write and analyse list-processing functions.

What is Functional Programming?

Functional programming (FP) is a programming paradigm in which computation is expressed as the evaluation of mathematical functions. It contrasts sharply with imperative programming.

Pure Functional Programming
A style where programs consist entirely of function definitions and function applications. There are no assignment statements, no mutable state, and no side effects. Every function is referentially transparent (see Chapter 7).

Mathematical functions

A mathematical function is a mapping from a set of inputs to a single output. It is purely determined by its inputs — calling it with the same arguments always returns the same result. This is what referential transparency means.

Contrast: in imperative languages, a function may read a global variable, modify state, or perform I/O — its result may differ between calls with identical arguments.

Advantages of the functional model

  • Easier to reason about: no side effects means substituting equals for equals (equational reasoning) is always valid.
  • Easier to test: pure functions are fully determined by their inputs — no need to set up mutable state.
  • Parallelism: pure functions can be evaluated in any order or concurrently without race conditions.
  • No "off-by-one" loop errors: recursion replaces iteration — base case and recursive case clearly define the computation.

Scheme — Overview

Scheme is a minimalist, lexically-scoped dialect of LISP developed in the mid-1970s. Key characteristics:

  • S-expression syntax: all code is expressed as parenthesised lists (S-expressions).
  • Functions are first-class: functions can be assigned to variables, passed as arguments, returned as values.
  • Lexical scoping: variables are resolved in the static environment where the function is defined, not where it is called.
  • Tail-call optimisation: properly tail-recursive implementations must optimise tail calls, enabling recursion without stack overflow.
  • Dynamic typing: types are associated with values, not variables; no type declarations.

In this course, you use DrRacket with the Scheme language (#lang r5rs or similar). All your programs must use the textbook's Scheme subset.

Syntax: S-Expressions

Everything in Scheme is either an atom or a list:

  • Atom: a number (42, 3.14), a symbol (foo, +), a Boolean (#t, #f), a string ("hello").
  • List: (element1 element2 ...) — any number of elements between parentheses.

A function call is written as a list where the first element is the function and the remaining elements are arguments. This is prefix notation:

(+ 3 4)         ; => 7
(* 5 6)         ; => 30
(+ 1 2 3 4)     ; => 10   (+ accepts any number of args)
(> 5 3)         ; => #t   (true)
(= 5 5)         ; => #t

Everything is an S-expression — data and code have the same form. This is what LISP/Scheme mean by "code is data".

Special Forms

Special forms look like function calls but are not evaluated like them — they have their own evaluation rules defined by the language.

DEFINE

Binds a name to a value or function in the current environment.

; Binding a variable
(define pi 3.14159)
(define name "Alice")

; Defining a function
(define (square x)
  (* x x))

(square 5)  ; => 25
(square 3)  ; => 9

; Alternative (using lambda implicitly):
(define square (lambda (x) (* x x)))

The function definition form (define (name params...) body) is syntactic sugar for (define name (lambda (params...) body)).

IF and COND

; IF — two-way selection
(if condition  then-expression  else-expression)

(if (> x 0)
    "positive"
    "non-positive")

; COND — multiple selection (like case/if-else-if)
(cond
  ((= x 0) "zero")
  ((> x 0) "positive")
  (else    "negative"))  ; else clause handles remaining cases

Both if and cond are expressions in Scheme — they return a value. The else branch of if is required when the if is used for its value.

LET

Creates local variable bindings within a lexical scope.

; LET — bind local variables; bindings are parallel (all rhs evaluated first)
(let ((x 5)
      (y 3))
  (+ x y))   ; => 8   (x and y only exist inside the let body)

; LET* — sequential bindings (each binding sees the previous ones)
(let* ((x 5)
       (y (* x 2)))     ; y can reference x because let*
  (+ x y))              ; => 15

Predicates and List Operations

A predicate is a function that returns a Boolean (#t or #f). By convention, predicate names end with ?.

(null? '())         ; #t  — is the list empty?
(null? '(1 2))      ; #f
(pair? '(1 2))      ; #t  — is this a non-empty list (a pair)?
(pair? '())         ; #f
(number? 42)        ; #t
(symbol? 'foo)      ; #t
(equal? '(1 2) '(1 2))   ; #t  — structural equality

Core list operations

; car — first element of a list
(car '(1 2 3))   ; => 1

; cdr — rest of the list (everything except the first element)
(cdr '(1 2 3))   ; => (2 3)
(cdr '(x))       ; => ()    (empty list)

; cons — construct: prepend an element to a list
(cons 1 '(2 3))   ; => (1 2 3)
(cons 1 '())      ; => (1)
(cons 1 (cons 2 (cons 3 '())))  ; => (1 2 3)

; list — construct a list from elements
(list 1 2 3)      ; => (1 2 3)

; Quote prevents evaluation
'(1 2 3)          ; the list (1 2 3) as data, NOT a function call
The quote notation

'(1 2 3) is shorthand for (quote (1 2 3)). It tells Scheme to treat the expression as data (a list), not as a function call. Without quote, (1 2 3) would try to call 1 as a function — an error.

Tail Recursion

Tail Call
A function call is a tail call if it is the last operation performed by the calling function — the calling function does nothing with the return value except return it directly.
Tail Recursion Optimisation (TCO)
When a recursive call is a tail call, the Scheme implementation is required to reuse the current stack frame for the recursive call (instead of creating a new one). This allows recursion to run in constant stack space — equivalent to a loop. Without TCO, deeply recursive functions would overflow the stack.

Non-tail-recursive vs tail-recursive

; Non-tail-recursive factorial
; The multiplication (* n ...) happens AFTER the recursive call returns
; → NOT a tail call → stack grows with n
(define (factorial n)
  (if (= n 0)
      1
      (* n (factorial (- n 1)))))    ; recursive call is NOT in tail position

; Tail-recursive factorial using an accumulator
(define (factorial n acc)
  (if (= n 0)
      acc                            ; base case: return accumulator
      (factorial (- n 1) (* n acc)))) ; tail call — nothing done after return

(factorial 5 1)  ; => 120

The tail-recursive version passes the accumulated result as a parameter, so the recursive call is the last thing done — a true tail call that can be optimised.

Higher-Order Functions

Higher-Order Function
A function that takes one or more functions as arguments, or returns a function as its result (or both).

APPLY

apply calls a function with a list as its argument list:

(apply + '(1 2 3 4))     ; => 10  (equivalent to (+ 1 2 3 4))
(apply max '(3 1 4 1 5)) ; => 5

MAP

map applies a function to every element of a list and returns a new list of results:

(map (lambda (x) (* x x)) '(1 2 3 4 5))
; => (1 4 9 16 25)

(map car '((1 a) (2 b) (3 c)))
; => (1 2 3)   — car of each sublist

List Processing Functions

⚠ Exam Focus — Know These Functions Thoroughly

The exam implementation question will be at most as complex as member, equalsimp, equal, and append. Study each function until you can write it from scratch and trace its execution step by step.

member — Check list membership

Returns #t if atm is a member of list lst, #f otherwise. Assumes a flat list of atoms.

(define (member atm lst)
  (cond
    ((null? lst) #f)                   ; empty list — not found
    ((equal? atm (car lst)) #t)        ; found at head
    (else (member atm (cdr lst)))))    ; recurse on tail

; Tests:
(member 'b '(a b c))    ; => #t
(member 'x '(a b c))    ; => #f
(member 3 '(1 2 3 4))   ; => #t
(member 3 '())           ; => #f

Trace of (member 'b '(a b c))

; Call 1: atm='b, lst='(a b c)
;   (null? '(a b c)) => #f
;   (equal? 'b (car '(a b c))) = (equal? 'b 'a) => #f
;   Recurse: (member 'b '(b c))
; Call 2: atm='b, lst='(b c)
;   (null? '(b c)) => #f
;   (equal? 'b (car '(b c))) = (equal? 'b 'b) => #t
;   Return #t

equalsimp — Equality of simple lists

Tests whether two lists of atoms are equal (element by element). Does not handle nested lists.

(define (equalsimp lst1 lst2)
  (cond
    ((null? lst1) (null? lst2))           ; both empty => #t; one empty => #f
    ((null? lst2) #f)                     ; lst1 not empty but lst2 is
    ((equal? (car lst1) (car lst2))
     (equalsimp (cdr lst1) (cdr lst2)))   ; heads equal, compare tails
    (else #f)))                           ; heads not equal

; Tests:
(equalsimp '(a b c) '(a b c))   ; => #t
(equalsimp '(a b) '(a b c))     ; => #f
(equalsimp '() '())              ; => #t
(equalsimp '(1 2) '(1 3))       ; => #f

equal — Equality including nested lists

Tests structural equality of any two S-expressions, including nested lists.

(define (equal? lst1 lst2)
  (cond
    ((not (pair? lst1)) (eqv? lst1 lst2))    ; both atoms: compare directly
    ((not (pair? lst2)) #f)                   ; lst1 is pair but lst2 is atom
    ((equal? (car lst1) (car lst2))
     (equal? (cdr lst1) (cdr lst2)))          ; compare heads, then tails
    (else #f)))

; Tests:
(equal? '(a (b c) d) '(a (b c) d))  ; => #t  (nested list)
(equal? '(a (b c)) '(a (b x)))      ; => #f
(equal? 'a 'a)                       ; => #t  (atoms)
(equal? '() '())                     ; => #t

Key difference from equalsimp: equal recurses into nested lists ((car lst1) might itself be a list), while equalsimp only compares flat lists of atoms.

append — Concatenate two lists

Returns a new list that is the concatenation of lst1 and lst2.

(define (append lst1 lst2)
  (if (null? lst1)
      lst2                                    ; base: nothing to prepend
      (cons (car lst1)
            (append (cdr lst1) lst2))))        ; prepend head, recurse on tail

; Tests:
(append '(a b) '(c d))    ; => (a b c d)
(append '() '(1 2))        ; => (1 2)
(append '(1 2) '())        ; => (1 2)
(append '(a) '(b c d))     ; => (a b c d)

Trace of (append '(a b) '(c d))

; (append '(a b) '(c d))
;   lst1 = '(a b), not null
;   (cons 'a (append '(b) '(c d)))
;     lst1 = '(b), not null
;     (cons 'b (append '() '(c d)))
;       lst1 = '(), null → return '(c d)
;     → (cons 'b '(c d)) = '(b c d)
;   → (cons 'a '(b c d)) = '(a b c d)
; Result: '(a b c d)

Additional example: reverse

Using append, we can write a simple (non-tail-recursive) reverse:

(define (my-reverse lst)
  (if (null? lst)
      '()
      (append (my-reverse (cdr lst))
              (list (car lst)))))

(my-reverse '(1 2 3 4))  ; => (4 3 2 1)

Chapter 15 Summary

  • Functional programming: computation via function application, no assignment, no mutable state, referential transparency.
  • S-expressions: atoms or lists. Function call = list with function as first element (prefix notation).
  • quote: prevents evaluation. '(1 2 3) is data, not a call to function 1.
  • DEFINE: bind a name to a value or function.
  • IF: two-way selection expression. COND: multi-way selection expression.
  • LET: parallel local bindings. LET*: sequential (each binding sees previous ones).
  • car: first element. cdr: rest. cons: prepend. null?: empty list test.
  • Tail recursion: recursive call is the last operation; Scheme optimises to constant stack space.
  • APPLY: call function with a list as argument list. MAP: apply function to each element, return list of results.
  • member: flat list membership test. equalsimp: flat list equality. equal: nested list equality. append: list concatenation.
